# 변경 사항 안내 (Changelog)

이 프로젝트의 모든 주요 업데이트 내역을 기록하는 파일입니다.

## [1.1.0] - 2026-04-22

### 🆕 추가된 기능
- **화면 구성 확장:** 총 4개 화면 완성 (메인 / A채널 상세 / B채널 상세 / 시스템 상태)
- **화면 밝기 조절:** 10% 단위로 조절 가능, 버튼을 꾹 누르면 연속 조절, 조작 없을 시 자동 저장 및 복귀
- **시스템 설정 모드:** 화면에서 직접 CPU 속도, WiFi/블루투스 전원, 밝기를 선택하고 변경 가능
- **설정 기억 기능:** 전원을 껐다 켜도 밝기, CPU 속도, WiFi/BT 설정값이 유지됨
- **통신 자동 복구:** WiFi 신호 간섭 감지 시 자동으로 연결을 복구하는 기능 추가
- **진단 로그 강화:** 연결 끊김이나 재연결 과정을 시리얼 로그로 더 상세히 표시
- **코드 구조 개선:** 화면 UI 관련 코드를 별도 파일(`ui.h`, `ui.cpp`)로 분리하여 관리 효율 증대
- **설명서 보완:** README에 시스템 구조도(그림)와 초보자를 위한 쉬운 해설 섹션 추가

### 🔄 개선된 사항
- **버튼 조작 방식 정리:** 살짝 누르기(화면 전환), 꾹 누르기(설정 진입/실행)로 조작 체계 단순화
- **버튼 위치 수정:** 보드 실제 위치에 맞춰 버튼 매핑 수정 (위쪽: GPIO14, 아래쪽: GPIO0)
- **시간 표시 형식:** 시스템 가동 시간을 보기에 편한 '시:분:초' 형식으로 통일
- **발열 관리 최적화:** 백라이트 제어를 16단계로 세분화하고, 기본 CPU 속도를 저발열 모드(80MHz)로 조정
- **노이즈 대응:** 실제 차량 환경의 전기적 노이즈로 인한 통신 오류 방지 로직 강화

### 🛠️ 수정된 문제 (버그 픽스)
- **무한 재부팅 방지:** WiFi와 블루투스를 동시에 켰을 때 전력이 부족하여 재부팅되던 현상 완화
- **신호 감지 개선:** 짧은 순간 신호가 끊겼을 때 '신호 없음' 메시지가 너무 자주 뜨던 예민함 조절
- **밝기 조절 안정화:** 밝기 변경 직후 화면이 깜빡이거나 조작이 안 되던 버그 수정

## [1.0.0] - 2026-04-22

### 🆕 초기 기능
- **테슬라 모니터링 기본:** ESP-NOW 통신을 이용한 테슬라 CAN 데이터 수신 기초 설계
- **3페이지 UI:** 메인 및 각 채널별 상세 정보 화면 구현
- **화면 전환:** 물리 버튼(GPIO 0, 14)을 이용한 페이지 넘기기 기능
- **상태 표시:** 신호 없음(NO SIGNAL), 주행 보조(Nag) 모드 상태 표시
- **통신 진단:** CAN 통신 상태 및 오류(BusOff) 여부 표시
- **데이터 분석:** 테슬라의 최신 데이터 형식(52바이트) 및 기존 형식(22바이트) 모두 지원
